# Product Requirements

## Purpose
Tenant Mix Dashboard supports senior retail-real-estate analysts, leasing directors, shopping-centre managers and asset managers in competitive tenant-mix analysis.

## Core user questions
1. How does the focus object compare with a selected group?
2. Which categories are above or below the group median?
3. Which brands are unique, intersecting or absent from the focus object?
4. Which comparison objects are most similar by tenant mix?
5. Which conclusions are supported by data and which require additional commercial assessment?

## Primary sections
- Overview: concise executive position and signals.
- Comparability: object-level benchmarking.
- Categories: category structure, medians and density.
- Brands: aggregated brand registry and sources.
- Upcoming openings: confirmed future changes.
- Data quality: coverage, freshness and validation.
- History: only when multiple comparable snapshots exist.

## Product principles
- Accuracy before visual novelty.
- One calculation context for all dependent blocks.
- Facts, signals and recommendations must be visually and verbally distinct.
- Global filters define the analysis scope; local table filters do not silently alter global KPI.
- Every non-obvious metric must expose methodology and scope.
- Missing data is shown as `н/д`, never silently replaced.

## Out of scope without additional data
- Lease-economics recommendations.
- Revenue or NOI forecasts.
- Brand deal readiness.
- Store-size fit.
- Historical trends from a single snapshot.
- Causal conclusions from correlation alone.
