# Tenant Mix Online Dashboard

Интерактивный дашборд для сопоставления арендного состава торговых центров России. Он позволяет сравнивать объекты по масштабу, географии, категориям, уникальным и пересекающимся брендам, а также просматривать реестр арендаторов и актуальные анонсы будущих открытий.

Рабочая версия: [GitHub Pages](https://koptelov-vk.github.io/tenant-mix-dashboard/)

## Что показывает дашборд

### Выбор сопоставимых объектов

Объекты можно одновременно фильтровать по общей площади комплекса и географии.

Классификация по GBA:

- Районные: 7 500–50 000 м².
- Региональные: 50 000–120 000 м².
- Суперрегиональные: от 120 000 м².
- Нет данных по GBA/GLA.

Географические группы:

- Москва.
- Санкт-Петербург.
- Нижний Новгород.
- Регионы без Москвы и Санкт-Петербурга.

По умолчанию выбраны суперрегиональные объекты Нижнего Новгорода и других регионов. Фантастика отмечена флажком на сравнительных шкалах.

### Основные показатели

- **Выбрано объектов в срезе** — число ТЦ, соответствующих текущим фильтрам.
- **Выбрано городов в срезе** — число городов выбранных объектов.
- **Арендаторов в срезе** — число строк арендаторов выбранных ТЦ.
- **Брендов в срезе** — число различных нормализованных брендов.

Каждая KPI-плашка имеет собственный Excel-подобный фильтр с поиском, мультивыбором и командами «Выбрать все» / «Снять все». Фильтры объектов и городов уточняют состав сравниваемых ТЦ, а фильтры арендаторов и брендов применяются к строкам данных во всех аналитических блоках.

### Аналитические блоки

1. **Уникальные бренды по категориям.** Бренд представлен только в одном объекте текущего среза.
2. **Пересекающиеся бренды по категориям.** Бренд представлен минимум в двух выбранных объектах.
3. **Общее количество арендаторов в категориях.** Категории отсортированы по числу брендов в текущем срезе.
4. **Реестр брендов.** Детальные строки с Excel-подобными фильтрами по бренду, характеристике, категории, объекту и источнику.

Шкалы блоков 1–2 рассчитываются отдельно для каждой категории по минимальному и максимальному значениям текущего среза. Маркеры показывают абсолютное число брендов, а флажок выводит выбранный объект на передний план.

На мобильных устройствах блок 4 отображается карточками. Фильтры остаются доступны над реестром, а подтверждённые URL в колонке «Источник» открываются в новой вкладке.

## Логика подготовки данных

1. Каталоги арендаторов загружаются с официальных сайтов и доступных структурированных источников.
2. Коммерческие разделы проверяются отдельно: магазины, еда, услуги, развлечения, интерьерные форматы и «скоро открытие».
3. Инфраструктура ТЦ не включается в арендаторов: Wi-Fi, парковка, стойка информации, навигация и обслуживание маломобильных посетителей.
4. Названия брендов очищаются от юридических форм, адресных хвостов, товарных префиксов и вариантов написания.
5. После нормализации удаляются дубли внутри одного объекта.
6. Категория и характеристика уникальности пересчитываются по нормализованному бренду.
7. Дашборд собирается заново и проходит автоматические проверки.

Изменения в данных должны иметь источник и статус подтверждения. Непроверенные сведения не должны заменять подтверждённые без явного основания.

## Пересборка

Для локальной пересборки используются Python и Node.js с Playwright:

```powershell
python scripts/normalize_all_brands.py
python scripts/audit_brand_normalization.py
python scripts/build_interactive_tenant_dashboard.py
node scripts/test_table_filters.mjs
node scripts/test_new_malls.mjs
```

Результат создаётся в `outputs/tenant-mix-online-dashboard.html`. HTML автономен и содержит данные, стили и логику фильтрации внутри одного файла.

## Контроль качества

Перед публикацией проверяются:

- отсутствие пустых брендов и нормализованных ключей;
- отсутствие дублей «объект + нормализованный бренд»;
- соответствие признака «Уникальный/Пересекающийся» фактическому присутствию бренда;
- стабильность размеров колонок и работа мультивыбора в фильтрах;
- отсутствие ошибок JavaScript;
- мобильная компоновка без горизонтального переполнения;
- корректность ссылок источников;
- отсутствие необъяснимого сокращения каталога объекта более чем на 10%.

Текущий аудит нормализации: 0 дублей, 0 пустых названий, 0 пустых ключей и 0 ошибок уникальности. Пара `XC` / `ХС` сохранена раздельно как два разных бренда.

## Автоматическое обновление

В Codex настроена еженедельная сверка каждый понедельник в 02:00 по московскому времени. Задача:

- проверяет официальные сайты всех объектов;
- сверяет коммерческие разделы и новости;
- обновляет анонсы «Скоро открытие»;
- запускает нормализацию, аудит, сборку и браузерные тесты;
- сохраняет недельный отчёт и журнал изменений;
- публикует новую версию только после успешного контроля качества.

При ошибке предыдущая опубликованная версия сохраняется. Поскольку выполнение локальное, компьютер и Codex должны быть доступны в момент запуска.

Доступ к Beget хранится в локальном зашифрованном файле `.codex/secrets/beget-credential.xml`. Каталог исключён из репозитория и не должен передаваться другим пользователям.

## Ограничения

- Дашборд отражает снимок открытых источников, а не юридический реестр договоров аренды.
- Официальные сайты ТЦ обновляются с разной периодичностью и иногда содержат устаревшие карточки.
- Строки без подтверждённого URL источника отображаются без ссылки.
- GLA/GBA могут различаться между источниками; приоритет имеет наиболее надёжный источник, а расхождения сохраняются в аудите.
- Статус «Скоро открытие» требует регулярной повторной проверки после наступления заявленной даты.
